## 1. Introduction

## 2. Information We Collect

### 2.1 Location Data

The Application collects precise GPS coordinates (latitude, longitude, altitude, and accuracy radius) when a user initiates an emergency alert. Location data is:

- Collected at the moment of alert only (not continuously tracked)
- Included in the emergency packet transmitted to the pilot backend
- Stored in the backend database associated with the incident record
- Retained per the Data Retention Policy (see separate document)

**Purpose:** To provide first responders and pilot operators with the sender's location during an emergency event.

### 2.2 Device Telemetry

The Application collects:
- Battery level and charging state at time of alert
- Network connectivity status (cellular/Wi-Fi/offline)
- Network type
- Device operating system version (implicit via Expo SDK)

**Purpose:** To allow operators to assess device condition and connectivity when evaluating an alert.

### 2.3 Incident Information

- Incident type (selected by user from a predefined list)
- Additional notes (optional free text, maximum 500 characters)
- Timestamp of alert creation and transmission
- Retry count (number of times the system attempted to transmit the alert)

### 2.4 User and Device Identifiers

- A device-generated UUID assigned at first app launch (`deviceId`)
- A user-assigned identifier (`userId`) — during the pilot, this is the participant's employee ID or assigned pilot ID
- Application version

The Application does **not** collect names, Social Security Numbers, home addresses, financial information, or other sensitive personal identifiers.

### 2.5 Push Notification Token

An Expo push notification token is generated by the device when notification permission is granted. This token is stored on the incident record to enable status-update notifications to the user's device. It is an opaque device-specific token with no direct linkage to personal identity outside the platform.

### 2.6 Operator Actions

All actions taken by dashboard operators (viewing an incident, updating status, adding notes, changing assignments) are recorded in an immutable audit log. The log includes the operator's IP address and user-agent string.

## 3. How We Use This Information

| Data | Purpose |
|------|---------|
| GPS coordinates | Locate the individual who submitted the alert |
| Device telemetry | Assess device condition; support triage |
| Incident type and notes | Support pilot operator review |
| Device / user identifiers | Associate alerts with pilot participants for follow-up |
| Push token | Send status notifications to the individual who submitted the alert |
| Audit log | Accountability and record integrity for pilot evaluation |

We do **not** use collected information for advertising, profiling, commercial purposes, or any purpose outside the pilot program evaluation.

---

## 4. Data Storage and Security

- All data is stored in a PostgreSQL database hosted on Supabase (data center: AWS us-east-1)
- Data in transit is protected by TLS 1.2 or higher
- Emergency packets are encrypted with AES-256-GCM while queued on device
- Access to the backend database is restricted by connection string credentials and network access rules
- Backend API access is rate-limited and CORS-restricted to approved origins
- All material operator actions are logged in an immutable audit trail

See [SECURITY_OVERVIEW.md](SECURITY_OVERVIEW.md) for the full security posture.

---

## 5. Data Sharing

We do **not** sell, rent, or share personal information with third parties except as described below:

| Recipient | Purpose | Applicable Scenario |
|-----------|---------|---------------------|
| Supabase (Postgres hosting) | Database hosting only | All data storage |
| Vercel (dashboard hosting) | Web application hosting only | Dashboard access |
| Railway (API hosting) | API server hosting only | API requests |
| Pilot supervisors | Incident review during pilot | Alert submitted |
| Future: RapidSOS PULSE | Emergency data forwarding | Phase 3 only — not active |
| Future: CAD systems | Dispatch coordination | Phase 4 only — not active |

No data is shared with RapidSOS, CAD systems, or any other emergency infrastructure during the pre-pilot phase.
