# ER Offline SDK — Emergency System Disclaimer

**PLEASE READ CAREFULLY BEFORE USE OR DEPLOYMENT**

---

## THIS IS NOT A 911 SYSTEM

The ER Offline SDK is a **pre-pilot, experimental software prototype**. It is not a certified emergency communications system. It is not connected to any Public Safety Answering Point (PSAP), 911 center, Computer-Aided Dispatch (CAD) system, or any active emergency response infrastructure.

---

## CRITICAL LIMITATIONS

### No Guaranteed Emergency Response

Use of this application **does not guarantee** that emergency services will be dispatched, that any person will receive or review your alert in a timely manner, or that rescue or medical assistance will be provided. **This application does not replace calling 911.**

### No Real-Time Dispatch

This system does not route alerts to any dispatch center. Alerts submitted through this system go to a pre-pilot operator dashboard staffed by designated pilot participants only. It is not monitored 24/7. It is not monitored by law enforcement, fire, EMS, or any official emergency agency.

### Offline Queue Is Not Guaranteed Delivery

When a device is offline (no cellular or Wi-Fi signal), the emergency alert is stored in an encrypted queue on the device. Transmission only occurs when the device regains connectivity. **If the device loses power, is destroyed, is lost, or never regains connectivity, the alert may never be transmitted.**

The offline queue is a best-effort buffering mechanism, not a guaranteed delivery protocol.

### GPS Accuracy Is Not Guaranteed

Location data provided by this application depends on the device's GPS hardware, satellite visibility, and environmental conditions. In dense forest, underground, indoors, or in canyons, GPS coordinates may be inaccurate, stale, or unavailable. Users should always be prepared to communicate their location verbally if possible.

### Battery and Signal Metadata Is Informational Only

Battery level, signal status, and network type data are informational. This system does not monitor your device continuously. If your battery dies after submitting an alert, responders will not receive updates on your condition.

### No Guarantee of System Availability

This is a pre-pilot prototype system. It may be unavailable at any time due to maintenance, infrastructure failure, software updates, or service provider outages. No uptime guarantee is provided.

---

## ALWAYS CALL 911 FIRST

**In any life-threatening emergency, your first action should be to call 911 if you have any signal.** This application is intended to supplement — never replace — the 911 system.

If you have a signal, **call 911 now**.

If you have no signal and cannot call 911, this application may help you queue an alert for transmission when signal returns — but there is no certainty of response time or response at all.

---

## PERSONAL LOCATOR BEACONS AND SATELLITE COMMUNICATORS

For wilderness emergencies where no cellular signal is expected, we strongly recommend carrying a **Personal Locator Beacon (PLB)** registered with NOAA, or a satellite communicator such as Garmin inReach or SPOT. These devices transmit directly to rescue coordination centers via satellite regardless of cellular coverage and are the current gold standard for remote emergency signaling.

This application is not a replacement for a PLB or satellite communicator.

## PILOT PROGRAM CONTEXT

During the pre-pilot period, this application is being used exclusively for **data collection and system evaluation** purposes. Any alerts submitted during the pilot are reviewed by designated pilot supervisors, not emergency dispatchers. Alerts do not trigger any emergency response.

Pilot participants have been informed of these limitations in writing through the pilot onboarding agreement.
